    redis从海量数据里查询

    ​ redis生成海量数据的方法：

    ​

    • keys pattern** : 查找所有符合给定模式pattern的key

      1. 一次性返回所有的key

      2. 当数据量过大时，会卡顿

    • scan cursor [match pattern] [count num]

      1. 基于游标的迭代器，需要基于上一次游标延续之前的迭代过程

      2. 以0作为游标开始一次新的迭代，知道命令返回游标0完成一次遍历

      3. 并不保证每次执行都返回给定数量的元素，支持模糊查询

      4. scan每次查询完会返回本次查询的最后key的游标，即用于下一次查询
